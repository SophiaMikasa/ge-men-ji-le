1. 项目名称：​ 哥们记了

2. 系统定位：

这是一个专注于学生群体的本地化桌面应用，用于自动化分析个人微信/支付宝账单，并通过交互式图表提供深度消费洞察。其核心价值在于“免记账、护隐私、深洞察”。

3. 独特设计与差异化（对抗雷同的关键）：

深度场景化智能分类引擎：​ 与通用记账软件不同，我们内置了“学生消费关键词库”（如：食堂、校园卡、教育超市、社团费、学苑餐厅、校内快递等），能极高精度地自动归类消费记录，无需或极少需要用户手动调整。

洞察驱动的交互式可视化：​ 不仅展示静态饼图、柱状图，我们将使用 ECharts​ 实现交互式图表。用户可点击图表元素（如饼图的某个分类区块）下钻查看该分类下的具体消费明细，实现从宏观概览到微观分析的无缝过渡。

坚定的本地化与隐私安全：​ 所有数据处理均在用户本地计算机上完成，数据绝不上传至任何服务器。这是相对于“鲨鱼记账”、“随手记”等云存储应用的核心优势，特别符合学生对隐私安全的敏感需求。

二、 预期用户量

基于团队所在高校的实际情况，我们设定以下预期：

初期目标（第一版本发布后6个月内）：​ 覆盖团队所在学院及友好社团，预计达到 200名​ 活跃学生用户。

中期目标（一年内）：​ 通过校园论坛、社交平台推广，覆盖全校，预计达到 1000名​ 活跃学生用户。

推广途径：​ 通过学生会、社团联合会等组织进行内部推荐；在校园BBS、表白墙等平台发布体验版。

三、 需求规格说明书（SRS）核心内容框架

以下是您撰写正式文档的提纲，请在此基础上扩充细节。

1. 引言

1.1 项目背景与目标：​ 简述学生群体面临的财务管理困境和现有软件的不足。

1.2 文档范围：​ 本文档定义了“哥们记了”v1.0版本的功能与非功能需求。

1.3 目标读者：​ 项目团队成员、指导教师。

2. 总体描述

2.1 产品愿景：​ 成为学生群体首选的、安全、智能、易用的个人财务分析助手。

2.2 用户特征：

核心用户：​ 高校在校学生（本科、研究生）。

用户特征：​ 消费场景固定（食堂、校内超市、网购），对价格敏感，注重隐私，缺乏坚持手动记账的毅力，渴望了解自身消费习惯。

2.3 系统真实性、可用性与价值：

真实性：​ 需求源于团队成员及身边同学的真实痛点，已成功完成支付宝官方CSV账单的解析原型，技术路线（Python+Pandas+Flask+ECharts）可行。

可用性：​ 图形化界面操作，只需“导出账单-拖拽上传-查看结果”三步，无需任何会计或技术知识。

价值：​ 提升用户的财务感知能力，通过数据可视化帮助用户发现不良消费习惯，从而促进理性消费，为培养理财素养奠定基础。

3. 功能性需求

FR1: 数据导入与清洗模块

FR1.1：系统应支持上传支付宝、微信支付导出的标准CSV格式账单文件。

FR1.2：系统应能自动解析文件，提取交易时间、金额、对方名称等关键字段。

FR1.3：系统应能过滤无效记录（如退款、转账）。

FR2: 智能分类与聚合模块

FR2.1：系统应基于内置的学生消费关键词库，对消费记录进行自动分类（如：餐饮、购物、交通等）。

FR2.2：用户可对自动分类结果进行手动修正和重分类。

FR2.3：系统应能按类别、月度、年度对消费金额进行聚合计算。

FR3: 多维度可视化分析模块

FR3.1：系统应以饼图形式展示指定时间段内的消费结构分布。

FR3.2：系统应以折线图/柱状图展示月度消费总额的趋势变化。

FR3.3：系统应支持交互操作，如点击饼图区块可下钻查看该类别下的明细列表。

FR4: 洞察与报告模块

FR4.1：系统应能生成简单的月度消费报告（PDF格式）。

FR4.2：系统应能提供关键洞察，如“本月餐饮支出环比上涨20%”。

4. 非功能性需求（技术需求）

性能需求：​ 导入并解析1000条消费记录的时间应小于3秒。

安全性需求：​ 所有数据在处理和存储过程中均位于用户本地设备，系统绝不申请网络访问权限。

可用性需求：​ 界面简洁直观，主要操作流程应在3步内完成。

兼容性需求：​ 桌面端应用，优先支持 Windows 10/11 及 macOS。

5. 约束与假设

约束：​ 依赖支付宝/微信官方导出的账单格式，若格式更新，程序需相应调整。

假设：​ 用户导出的账单文件是真实、完整的。
